WBL Data Pipelines

Four layers. The CLI sync fetches data from external sources, computes everything, and writes the canonical answers to Supabase. The browser is a pure reader — it reads precomputed values from Supabase for current-year data. The browser only recomputes when custom scouting is active or when viewing historical years.

External Data Sources — "Where does the raw data come from?"

CLI Sync Reads (sync-db.ts)

WBL JSON API (authenticated) /api/players, /api/teams, /api/playerpitchstats, /api/playerbatstats, /api/contract, /api/scout (OSA scouting), /api/drs, /api/date. Auth: x-api-key header.
WBL Firebase (public) teamHome/{teamId}/injuries — active injury data (player, duration, injury type). Primary source for injury_days_remaining.
WBL CSV (public, fallback) /csv/players.csv — fallback injury source if Firebase unavailable. Fields: injury_dl_left, injury_left. Less current than Firebase.

Browser Reads Directly

WBL API: /api/date Game date + season. Used by DateService for year detection.
WBL API: /api/scout (custom scouting only) Team-specific scouting ratings (paginated, 2000/page). Only fetched when user triggers custom scouting import. Stored in IndexedDB. Sets hasCustomScouting flag.
Nothing else. All stats, players, contracts, OSA scouting, injuries come from Supabase. The browser never calls /api/players, /api/playerpitchstats, etc.
CLI Sync (sync-db.ts) — fetches, computes, writes canonical answers

Pipeline Steps

Step 1-3: Fetch + Write Raw Data Teams, players (+ injury_days_remaining from Firebase/CSV), stats, contracts, OSA scouting → Supabase tables. Patch IC player levels. Build SyncContext.
Step 4: Compute TR Pitcher + hitter True Ratings via shared HitterTrueRatingsCalculationService. Stored in player_ratings table. Reused by all subsequent steps.
Step 5: Compute TFR Pitcher + hitter True Future Ratings (peak projections for prospects). Stored in player_ratings + precomputed_cache (tfr_prospects arrays). Includes peak WAR, peak IP/PA, peak rates.
Step 5.5: Compute Projections Current-season projections for all MLB-ready players. Uses Step 4 canonical TR. Applies ALL adjustments: injury (days remaining → PA/IP reduction), park factors, aging, defensive value, SB blending. This is the canonical projection. Browser reads it, does not recompute.
Step 6: Build Lookups Scouting, contract, DOB, defensive, league_context lookups. Compact JSONB for fast browser reads (~7 requests total).

Supabase Output (canonical cache)

player_ratings pitcher_tr, hitter_tr, pitcher_tfr, hitter_tfr per player. Browser reads for TR badges, rating bars, TFR upside.
precomputed_cache: projections pitcher_projections, batter_projections — full projection arrays with WAR, PA/IP, rates, park factors, injury adjustment, defensive value. Every view that shows projected WAR/PA/IP reads from here.
precomputed_cache: TFR prospects pitcher_tfr_prospects, hitter_tfr_prospects — peak projections for prospects. Used by Farm Rankings, Draft Board, Team Planning, modal peak mode.
precomputed_cache: lookups scouting, contract, DOB, defensive_lookup, league_context.
players table injury_days_remaining column — set by sync from Firebase/CSV. Read by modal for injury badge display.
Browser: Read Precomputed — the default path for all current-year data

What the browser reads

Current-year projections (WAR, PA, IP) BatterProjectionService + ProjectionService read from precomputed_cache. All views that show projected WAR/PA/IP consume these services. No independent recomputation. The CLI answer IS the answer.
Canonical TR TrueRatingsService reads from player_ratings (fast path) or computes from stats + scouting (when custom scouting active).
Peak projections (TFR) TeamRatingsService reads tfr_prospects from precomputed_cache. Used for Farm Rankings, Draft Board, Team Planning, modal peak toggle.
Player injury status PlayerService reads injury_days_remaining from players table. Used by modals for injury badge. Projection impact already baked into cached projections.

All consumer views

Profile Modals (current mode) WAR badge + projection row read precomputed projection for current year. Modal resolves canonical TR for rating bars/percentiles.
Projections View Reads precomputed batter + pitcher projection arrays directly.
Team Ratings (all tabs) Power Rankings, WAR Projections, Win Projections, Standings — all read precomputed projections for per-player WAR/IP/PA.
Trade Analyzer Player valuation uses precomputed projection WAR.
Farm Rankings / Draft Board Prospect rankings read precomputed TFR + peak projections.
Team Planning Roster construction reads precomputed projections + TFR for peak values.
Global Search Search results show TR + TFR from precomputed ratings.
Browser: Recompute — only these specific cases

When the browser recomputes

Custom scouting active (hasCustomScouting = true) User uploaded their own scouting ratings. Precomputed cache is invalid for affected players. Browser recomputes TR → TFR → projections using merged my > osa scouting. All fast paths bypassed.
Historical year selected No precomputed cache for past years. Browser computes projections from historical stats. Year dropdown > 1 year from current = recompute.
Backtesting mode (ProjectionsView) Comparing projections vs actual stats for a completed season. Requires computing projections from pre-season data.

Shared computation services (used by recompute paths)

HitterTrueRatingsCalculationService Pure math: stat blending, scouting regression, wOBA. Shared import between CLI + browser. Same code, same results.
ModalDataService computeBatterProjection / computePitcherProjection — pure functions for projection math. Used by CLI Step 5.5 AND browser recompute paths. Same code, same results.
ProjectionService / BatterProjectionService Orchestration: fetch data, call computation services, return results. Fast path: read precomputed cache. Slow path: recompute.

Invariants — these must always be true

  • One number, everywhere. A player's projected WAR/PA/IP must be identical across the projection table, modal WAR badge, modal projection row, team ratings, power rankings, and trade analyzer. If they differ, it's a bug.
  • Precomputed cache = canonical answer. For current-year data, the precomputed cache is the single source of truth. Views read it; they do not recompute independently. Adjustments (injury, park factors, defensive value, aging) are applied once in the CLI pipeline and baked into the cache.
  • Browser recompute = exception, not default. The browser only recomputes projections when custom scouting is active (hasCustomScouting flag) or when viewing historical years. No other path should trigger independent recomputation.
  • CLI = browser math. sync-db uses the same pure computation functions as the browser (shared imports). When the browser does recompute, it must produce identical results to the CLI for the same inputs.
  • New adjustments go in one place. Any new projection adjustment (injury, park factor, aging curve, etc.) is added to the CLI pipeline (sync-db Step 5.5). It automatically flows to all views via the precomputed cache. It is NOT added to individual views or modals.

Key Design Rules

  • Year dropdown = target season. User selects the season they want to see (e.g., 2022). Code internally derives the stats base year (2021). The user never sees or thinks about base years.
  • Custom scouting bypasses all fast paths. When hasCustomScouting is true, all precomputed Supabase caches are skipped and everything computes locally with merged my > osa scouting data.
  • WAR includes defensive value. Batter WAR = (wRAA + replacementRuns + sbRuns + defRuns + posAdj) / runsPerWin. Precomputed by CLI, cached in defensive_lookup.
  • Injury adjustment = PA/IP reduction. Active injury days (from Firebase) reduce projected PA/IP proportionally: ratio = max(0, 1 - injuryDays/180). Applied once in CLI Step 5.5. Threshold for visual indicator: 18+ days (≥10% impact).
  • Park factor adjustment. Batter: effective park factors by batter handedness. Pitcher: HR9 park factor. Applied once in CLI Step 5.5.
  • Canonical prospect ranking = peak projected WAR. Farm Rankings, Draft Board, Top 100 badge all sort by peak WAR desc from precomputed TFR.
